# STM32 智能小车项目指南

## 项目目标

用 STM32F103C8T6 完成一辆具备循迹、避障、测速和 OLED 状态显示能力的智能小车。项目面向零基础学习者，最终成果应能稳定演示 2-3 分钟，并能支撑课程报告或入门竞赛答辩。

## 默认硬件清单

| 硬件 | 用途 | 注意事项 |
| --- | --- | --- |
| STM32F103C8T6 最小系统板 | 主控核心 | 配合 ST-Link V2 下载和调试 |
| 两轮小车底盘 + 带编码器 TT 电机 | 运动平台 | 编码器用于测速和后续 PID |
| TB6612FNG 电机驱动 | 控制电机正反转和速度 | 电机电源和单片机电源必须共地 |
| 5 路 TCRT5000 循迹模块 | 识别黑线位置 | 先用数字量，稳定后再优化 |
| HC-SR04 超声波模块 | 前方避障 | Echo 可能是 5V，需要分压保护 |
| 0.96 寸 I2C OLED | 显示运行状态 | 建议显示模式、速度、距离、偏差 |
| 电池盒、开关、杜邦线、洞洞板 | 供电和装配 | 线束要固定，避免小车运行时松动 |

## 四周实施路线

### 第 1 周：工具与基础

目标是跑通工具链，不追求功能复杂。

- 安装 STM32CubeIDE，或安装 STM32CubeMX + Keil。
- 用 ST-Link 下载第一个 LED 闪烁程序。
- 学会 GPIO 输入输出、定时器 PWM、串口输出。
- 用串口打印调试信息，例如 `printf("distance=%d\r\n", distance);`。
- 确认并购买硬件。

验收标准：能下载程序，LED 正常闪烁，串口能看到日志。

### 第 2 周：硬件单模块

目标是把关键硬件一个个调通。

- 装好底盘、电机、驱动板、电池和主控。
- 写 `Motor_SetSpeed(left, right)` 控制左右轮。
- 点亮 OLED，显示项目名和调试数据。
- 读取 HC-SR04 距离，并过滤明显异常值。

验收标准：电机能前进、后退、转向；OLED 能显示；距离读数稳定。

### 第 3 周：循迹与测速

目标是小车能沿黑线慢速运行。

- 读取 5 路循迹模块，形成黑线状态 bitmask。
- 根据传感器位置计算偏差值，例如 -2、-1、0、1、2。
- 用编码器统计左右轮脉冲，换算速度。
- 加入简单比例控制或 PID 雏形，减少跑偏。

验收标准：小车能沿黑线行驶，速度变化能在 OLED 或串口看到。

### 第 4 周：整合与答辩

目标是形成可展示版本。

- 整合自动循迹、避障停车、恢复运行逻辑。
- 调整基础速度、避障阈值、循迹阈值和转向增益。
- 准备固定演示路线，连续测试 2-3 分钟。
- 整理报告、流程图、接线图、代码结构说明和答辩话术。

验收标准：小车连续运行稳定，遇障碍能停车或绕行，报告材料完整。

## 软件模块划分

| 模块 | 建议接口 | 验收标准 |
| --- | --- | --- |
| 电机模块 | `Motor_SetSpeed(left, right)` | 左右轮方向正确，PWM 调速有效 |
| 循迹模块 | `Line_Read()`、`Line_GetError()` | 能稳定区分黑线和白底 |
| 避障模块 | `Obstacle_GetDistanceCm()` | 近距离触发停车，远距离正常运行 |
| 测速模块 | `Encoder_Update10ms()`、`Speed_GetLeftRpm()` | 速度随 PWM 变化 |
| 显示模块 | `Display_Update(status)` | 不接电脑也能判断状态 |

## 推荐状态机

```text
START
  |
  v
INIT -> SELF_CHECK -> RUN_LINE
                       |
                       | distance < threshold
                       v
                    OBSTACLE_STOP
                       |
                       | distance is safe
                       v
                    RUN_LINE
```

第一版可以只做停车避障：前方距离小于 18 cm 时停车，距离恢复后继续循迹。绕行可以作为增强功能。

## 默认引脚建议

| 引脚 | 外设模式 | 用途 |
| --- | --- | --- |
| PB6 / PB7 | TIM4 PWM | 左右电机 PWM |
| PB12-PB15 + PB5 | GPIO | TB6612 方向和 STBY |
| PA0 / PA1 | TIM2 Encoder | 左轮编码器 |
| PA6 / PA7 | TIM3 Encoder | 右轮编码器 |
| PA2 / PA3 / PA4 / PA5 / PA12 | GPIO Input | 5 路循迹输入 |
| PA8 / PA11 | GPIO / 定时测量 | HC-SR04 Trig / Echo |
| PB10 / PB11 | I2C2 | OLED SCL / SDA |
| PA9 / PA10 | USART1 | 串口调试 |

实际接线可以根据开发板引脚占用调整，但要在报告中保持“引脚表”和“代码定义”一致。

## 测试清单

- LED 闪烁和串口日志正常。
- PWM 占空比变化时电机速度明显变化。
- OLED 显示项目名、模式、速度、距离。
- 超声波 10-80 cm 距离读数稳定。
- 循迹模块能区分黑线和白底。
- 编码器速度随 PWM 增减同步变化。
- 小车能沿黑线运行。
- 小车遇到障碍物能停车或绕行。
- 连续演示 2-3 分钟不复位、不明显跑偏。

## 暂不纳入第一版的功能

- 视觉识别
- 蓝牙 App
- RTOS
- 复杂地图规划

这些功能可以作为后续增强，但第一版应优先保证循迹、避障和测速稳定。
